# Decision: Distribute macOS builds with Developer ID notarization

## Status

Accepted

## Date

2026-07-15

## Context

[[0004-unsigned-mvp-release|ADR-0004]] allowed the MVP to ship unsigned so release automation was not blocked by certificate/account setup. The project owner now has an Apple Developer account and wants users to install Tokengochi through a downloaded `.dmg` outside the Mac App Store.

macOS Gatekeeper treats unsigned downloaded apps as unverified. For a public download flow, Tokengochi needs a Developer ID Application signature and Apple notarization.

## Decision

- macOS release builds are signed with a Developer ID Application certificate.
- macOS release builds are notarized through App Store Connect API credentials during the GitHub Actions release workflow.
- The release process keeps using `.dmg` artifacts generated by Tauri, published through GitHub Releases.
- This supersedes ADR-0004 for macOS only. Windows remains unsigned until a Windows code-signing certificate is configured.
- Mac App Store distribution remains out of scope.

## Consequences

Positive:

- Users can install Tokengochi from a downloaded `.dmg` without the unsigned-app Gatekeeper workaround.
- The release path stays compatible with GitHub Releases and the existing Tauri updater.
- CI fails early if macOS signing/notarization secrets are missing or invalid.

Negative or tradeoffs:

- Apple certificate and App Store Connect API keys become production secrets that must be rotated carefully.
- Release builds now depend on Apple notarization availability.
- A real clean-machine download test is still needed for each notarized release because local artifacts do not preserve browser quarantine behavior.

## Alternatives Considered

- Keep shipping unsigned DMGs: simpler, but creates installation friction and trust issues for ordinary users.
- Mac App Store distribution: higher review/process overhead and not needed for direct download distribution.
- Manual local notarization: easier for a one-off release but brittle and hard to repeat.
