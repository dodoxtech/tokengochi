# Task: Pixel sprite renderer + pet behavior AI

## Context

Rendering approach in [[../../decisions/0003-canvas-sprite-rendering|ADR-0003]]. Builds on the proven overlay from [[0002-pet-overlay-window-spike|0002]].

## Goal

A living pet: pixel-art sprite that wanders the screen bottom, idles, sleeps, reacts to click/drag, with mood-driven animation sets.

## Scope

In scope: Canvas 2D sprite-sheet renderer (Aseprite JSON format, pixelated scaling); behavior state machine (idle/wander/sleep/dragged/react) with 30fps-active/2fps-idle ticking; placeholder Hatchling sprite set per the asset spec below; hit-testing → click-through toggling.

Out of scope: food/eating (0006), evolution forms (0009), final commissioned art.

## Art Style Guide

All sprites in this project follow one style so placeholder art can be swapped 1:1 for final art later.

- **Canvas size:** every pet frame is **32×32 px**. The character body occupies roughly 22–26 px; the remaining margin absorbs animation overshoot (hops, squash/stretch) so frames never clip.
- **Display scale:** rendered at **4× integer scale** (128×128 on screen) with nearest-neighbor (`imageSmoothingEnabled = false`, CSS `image-rendering: pixelated`). Never scale by non-integer factors.
- **Palette:** the **Sweetie 16** palette, and only it. Each sprite should use ≤ 8 of its colors. Reference hexes: `#1a1c2c #5d275d #b13e53 #ef7d57 #ffcd75 #a7f070 #38b764 #257179 #29366f #3b5dc9 #41a6f6 #73eff7 #f4f4f4 #94b0c2 #566c86 #333c57`. Store as `ui/assets/sprites/palette.gpl` (GIMP/Aseprite palette file).
- **Outline:** 1 px outline in `#1a1c2c` (near-black, never pure `#000`) around the full silhouette. Selective inner outlines only where limbs overlap the body.
- **Shading:** flat 2-tone — base color + one shadow tone from the palette, light source top-left. **No anti-aliasing, no gradients, no partial alpha** (alpha is 0 or 255 only), otherwise pixelated upscaling produces fringing on the transparent overlay.
- **Silhouette:** must read at 1× against both light and dark wallpapers (the outline guarantees this). Keep the design chunky; details smaller than 2×2 px vanish at 32×32.
- **Facing:** author **left-facing only**. The renderer mirrors with `ctx.scale(-1, 1)` for rightward movement — never draw a right-facing duplicate.
- **Hatchling character design (placeholder):** a round, chick-like blob hatched from an egg. Body `#ffcd75` (warm yellow) with shadow `#ef7d57`, lighter belly patch `#f4f4f4`, two dot eyes `#1a1c2c`, small `#ef7d57` cheek blushes, two stubby feet, a single leaf-like tuft (`#a7f070` / `#38b764`) on top. No arms.

## Asset Specification

### File formats

- **Runtime:** one packed **PNG sprite sheet (RGBA, transparent background)** + one **Aseprite JSON metadata file** (`--format json-array`, with frame tags) per sheet. The renderer consumes only these two files.
- **Source:** the editable `.aseprite` file is committed next to its exports so future agents/artists can regenerate.
- **Export command** (deterministic, re-runnable):

  ```bash
  aseprite -b hatchling.aseprite \
    --sheet hatchling.png --data hatchling.json \
    --format json-array --sheet-type rows --list-tags
  ```

  If placeholder frames are generated by script instead of Aseprite, the script must emit JSON that matches this exact schema (`frames[]` with `frame/duration`, `meta.frameTags[]` with `name/from/to/direction`).

### Folder layout

```
ui/assets/sprites/
├── palette.gpl                 # Sweetie 16 palette (single source of truth)
├── hatchling/                  # pet: one folder per evolution form
│   ├── hatchling.aseprite      # editable source
│   ├── hatchling.png           # packed sheet, 256×160 (8 cols × 5 rows of 32×32)
│   └── hatchling.json          # Aseprite JSON (array) with frameTags
└── effects/                    # overlay effects, shared across all pet forms
    ├── effects.aseprite
    ├── effects.png             # 16×16 frames
    └── effects.json
```

Future forms (0009) add sibling folders (`juvenile-owl/`, …); food skins (0006) add `food/`. Nothing outside `ui/assets/sprites/` may contain art.

### Asset types (2 kinds in this task)

1. **Pet sprite sheet** — the Hatchling, all body animations, 32×32/frame.
2. **Effects sheet** — small overlays drawn *on top of* the pet at anchored offsets, 16×16/frame: `zzz` (sleep bubble), `heart` (happy), `exclaim` ("!" on click), `dust` (landing puff after drag-drop).

The ground shadow under the pet is **procedural** (a semi-transparent ellipse drawn by the renderer), not an asset — it must squash independently of the sprite.

### Animation inventory (one complete movement = one tag)

Pet sheet — **31 frames total**:

| Tag     | Frames | Frame duration | Loop | Notes |
|---------|--------|----------------|------|-------|
| `idle`  | 4      | 150 ms         | yes  | 1 px breathing bob; blink on frame 3 |
| `walk`  | 6      | 100 ms         | yes  | 2-step cycle, 1 px vertical bounce |
| `sleep` | 4      | 250 ms         | yes  | eyes closed, slow body swell |
| `eat`   | 6      | 100 ms         | once | open mouth → chew ×2 → swallow gulp |
| `happy` | 6      | 100 ms         | ×2   | two small hops with squash/stretch |
| `drag`  | 2      | 150 ms         | yes  | dangling, feet wiggle, worried eyes |
| `react` | 3      | 100 ms         | once | startled squash on click, returns to idle |

Effects sheet — **10 frames total**: `zzz` 3 @300 ms loop, `heart` 2 @200 ms loop, `exclaim` 1 @400 ms once, `dust` 4 @80 ms once.

So the full deliverable is **41 frames across 2 PNG sheets + 2 JSON files + 1 palette file + 2 .aseprite sources**. A single movement is *done* when its tag exists in the JSON, every frame follows the style guide, and it loops/terminates cleanly in the renderer test page.
