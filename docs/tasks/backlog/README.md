# Product Backlog

This folder contains planned work that is not actively being implemented.

## Backlog Rules

- Use [[../../templates/task|Task Template]] for technical or project work.
- Use [[../../templates/user-story|User Story Template]] for user-facing product work.
- Add priority, estimate, dependencies, and acceptance criteria before moving a task to active.
- Search this folder before creating new tasks to avoid duplicates.

## Prioritization

Use simple priority labels:

- P0: urgent blocker
- P1: important next work
- P2: useful but not urgent
- P3: idea or someday item

## Recommended Delivery Order

Start with the lowest numbered task whose dependencies are complete. The number is the recommended sequence, while priority indicates importance.
